Agentic Loops

How stateless LLM calls combine with a programmatic loop to create the illusion of a reasoning agent.

KEY INSIGHT // The LLM has no memory between calls. It is the loop — written by you — that maintains state, decides what to do next, and creates the appearance of sustained reasoning.

Pseudo code
# ── State lives HERE, not inside the LLM ─────────────────────────

messages = [
  { role: "system",  content: "You are a helpful agent. Use tools when needed." },
  { role: "user",    content: user_input }
]

tools = [web_search, run_code, query_database]   # real functions


# ── The loop ──────────────────────────────────────────────────────

while True:

  # Safety valve — checked BEFORE the LLM call so it always fires
  if max_iterations_reached():
    handle_timeout()
    break

  # Every call replays the full history — LLM sees nothing else
  response = call_llm(messages, tools)


  # Branch on what came back
  if response.type == "text":

    show_to_user(response.text)
    break                        # agent is done


  elif response.type == "tool_call":

    tool   = find_tool(response.tool_name)
    result = tool.run(response.tool_args)  # real side-effect

    # Append both sides so the LLM sees them next iteration
    messages.append({ role: "assistant", content: response })
    messages.append({ role: "tool",      content: result })

    # ↑ This is where "memory" comes from — NOT from the LLM
    continue                     # go round again
What this means in practice
Costs scale with history
Every loop iteration sends the full conversation. Longer history = more tokens = more cost and latency. Summarisation strategies exist for long-running agents.
You own the stop condition
The LLM cannot stop the loop — only return text. Your code decides when to halt. Always set a max iteration guard to prevent runaway agents.
Tools are just functions
The LLM outputs a tool name and arguments as structured text. Your loop parses that and calls the real function. The LLM never executes anything directly.
Debugging is loop debugging
When an agent misbehaves, inspect the messages array at each step. The LLM responded to exactly what it was given — the answer is usually in the history.
